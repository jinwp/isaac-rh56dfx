# RH56DFX for Isaac Lab

This project integrates the Inspire Robots RH56DFX hand as an external Isaac Lab package.

## Layout

- `src/isaaclab_rh56dfx/robots/`: RH56DFX asset configs
- `src/isaaclab_rh56dfx/mdp/actions/`: custom mimic-aware action term
- `src/isaaclab_rh56dfx/tasks/play/`: manager-based play environments
- `src/isaaclab_rh56dfx/scripts/`: `random_agent`, `list_envs`, `build_usd`
- `tools/generate_urdf.py`: host-side xacro to URDF generation
- `data/urdf/`: generated left/right URDF files
- `data/usd/`: cached USD output generated by Isaac Sim

## Typical workflow

1. Generate left/right URDF files on the host by running `tools/generate_urdf.py --all` from this repository.

2. Start the Isaac Lab Docker environment with [`docker/isaaclab.mount.yaml`] mounted so this package is available as an external project inside the container.

3. Optionally prebuild USD assets inside the container:

```bash
/workspace/isaaclab/isaaclab.sh -p -m isaaclab_rh56dfx.scripts.build_usd --headless
```

4. Run random motion play:

```bash
/workspace/isaaclab/isaaclab.sh -p -m isaaclab_rh56dfx.scripts.random_agent \
  --task Isaac-RH56DFX-Left-Play-v0 \
  --num_envs 16
```

Right-hand variant:

```bash
/workspace/isaaclab/isaaclab.sh -p -m isaaclab_rh56dfx.scripts.random_agent \
  --task Isaac-RH56DFX-Right-Play-v0 \
  --num_envs 16
```

## Notes

- This package is designed for the user's git-cloned `IsaacLab/` workflow.
- It does not require modifying `IsaacLab/source`.
- The RH56DFX play tasks are instantiated as `ManagerBasedEnv` objects inside the script rather than through `gym.make(...)`.
- The RH56DFX temporary files previously placed in `isaac_so_arm101/` are now legacy entry points.
